## Разбор

Восемь из десяти пунктов — это два новых домена, которых нет в роадмапе:

1. **Персонализация** — сегменты аудитории, сбор поведенческих сигналов,
   рекомендации, триггерные пуши.
2. **Маркетинговый бюджет заведения** — кампании возврата, лимиты, атомарное
   списание, автостоп скидок.

### Ограничение по данным (на 21.07.2026)

| Сигнал | Есть | Комментарий |
|---|---|---|
| брони | 620 | единственный содержательный сигнал |
| избранное | 15 | практически пусто |
| позиции предзаказа | 243 | мало для рекомендаций блюд |
| просмотры, клики, сессии | 0 | не собираются вообще |

Вывод: персонализация сейчас построена быть не может — не на чем. Сначала сбор
сигналов, потом алгоритмы.

### Замечания по конкретным требованиям

- **Турист/местный.** Код страны SIM — гипотеза, а не факт. Хранить как сегмент с
  уверенностью, уточнять поведением (город, разброс городов в бронях, частота).
- **Возраст.** Хранить дату рождения, а не число: иначе данные протухают за год.
  Побочная выгода — повод для поздравления, лучший инфоповод для ресторана.
- **Флажок страны** на регистрации — лишнее поле, выводится из номера. Каждое
  поле режет конверсию регистрации.
- **Бюджет на скидки** — деньги: только тиыны, резервирование суммы в момент
  ВЫДАЧИ скидки (не использования), атомарное списание, жёсткий стоп, полная
  выписка операций. Логично делать после домена payments.
- **Веб-версия** уже существует (book-eat.com). Вопрос не «сделать», а «какой
  объём функций в вебе»: полноценная бронь с кабинетом или витрина.

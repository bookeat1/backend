# BookEat backend

Бэкенд платформы BookEat: брони, меню, заведения, платежи. Go, чистая архитектура,
PostgreSQL.

Этот файл про то, как поднять проект у себя на машине за десять минут.

---

## Что нужно установить

| Что | Версия | Зачем |
|---|---|---|
| Go | 1.25.7 или новее | сам проект |
| Docker | любой свежий | база данных |
| make | есть в macOS и Linux | короткие команды |

Проверить, что всё на месте:

```bash
go version      # go version go1.25.7 ...
docker version  # без ошибок
```

Если Go нет: на macOS `brew install go`, на Ubuntu удобнее скачать с go.dev, потому что в
репозиториях обычно версия старее.

## Запуск за пять шагов

### 1. Клонировать и подготовить настройки

```bash
git clone https://github.com/bookeat1/backend.git
cd backend
cp .env.example .env
```

### 2. Поднять базу

```bash
docker compose -f docker-compose.dev.yml up -d
```

Поднимутся две базы: рабочая на порту **5433** и отдельная для тестов на **5434**.
Порты нестандартные специально, чтобы не столкнуться с Postgres, который уже может стоять у
тебя локально.

### 3. Заполнить `.env`

Минимум, который нужно поправить руками:

```dotenv
DB_HOST=localhost
DB_PORT=5433
DB_DATABASE=bookeat
DB_USERNAME=bookeat
DB_PASSWORD=bookeat
DB_SSLMODE=disable

APP_ENV=development
APP_LOG_FORMAT=text     # читаемые логи вместо JSON (ключ уже есть в .env.example)

AUTH_JWT_PRIVATE_KEY=   # см. следующий шаг
```

**Ключ для токенов.** Приложение подписывает токены доступа RSA-ключом. Сгенерировать свой:

```bash
openssl genrsa 2048 | awk 'BEGIN{printf "AUTH_JWT_PRIVATE_KEY=\""} {printf "%s\\n", $0} END{print "\""}' >> .env
```

Ключ локальный, он нужен только твоей машине, никуда его отправлять не надо.

**Коды подтверждения.** В разработке SMS никуда не уходят: заглушка печатает код прямо в лог,
но только если `APP_ENV=development`. В любом другом окружении код в лог не попадает — это
сделано намеренно, одноразовый код в логах равен угону аккаунта.

### 4. Накатить схему

```bash
make migrate-up
```

Откатить последнюю миграцию, если понадобится: `make migrate-down`.

### 5. Запустить

```bash
make run      # HTTP API, по умолчанию на 0.0.0.0:8080
```

В отдельном окне, если нужны фоновые задачи (автоподтверждение броней, закрытие просроченных,
сверка платежей):

```bash
make worker
```

Проверить, что живо:

```bash
curl http://localhost:8080/health
# {"data":{"status":"ok"}}
```

Документация API: http://localhost:8080/docs (спека: /docs/openapi.yaml)

## Тесты

```bash
make test-short   # быстрые, без базы
make test         # все, включая интеграционные
```

Интеграционным нужна отдельная база, та самая на порту 5434:

```bash
export TEST_DATABASE_URL='postgres://bookeat:bookeat@localhost:5434/bookeat_test?sslmode=disable'
make test
```

**Важно:** прогонять тесты последовательно, флагом `-p 1`. Часть тестов броней чистит общие
таблицы и при параллельном запуске мешает друг другу. Это известная проблема, она в очереди
на починку:

```bash
go test -p 1 ./...
```

## Как устроен проект

```
cmd/            точки входа: http, worker, migrate, etl
internal/
  domain/       сущности и правила: брони, платежи, журнал проводок. Без зависимостей наружу
  usecase/      сценарии: создать бронь, провести платеж, обработать вебхук
  infrastructure/
    postgres/   работа с базой
    payment/    адаптеры эквайеров (FreedomPay, TipTopPay)
  transport/
    rest/       HTTP-обработчики и middleware
  bootstrap/    сборка приложения из частей
migrations/     схема базы, накатывается goose
docs/           спеки и разборы, включая платежи
deploy/         Dockerfile, compose и runbook для серверов
```

Правило зависимостей: `domain` не знает ни про базу, ни про HTTP. `usecase` знает про `domain`
и про интерфейсы, но не про их реализации. Всё конкретное живёт в `infrastructure`.

## Платежи

По умолчанию **выключены** (`PAYMENTS_ENABLED=false`). Так и должно быть локально: без ключей
эквайера они всё равно не работают, а случайно включённые платежи в разработке никому не нужны.

Ключи провайдеров в репозитории отсутствуют и не должны в нём появляться. Если понадобится
прогнать песочницу, ключи выдаются отдельно и кладутся в `.env`, который в гит не попадает.

## Если что-то не работает

**`make migrate-up` не видит базу.** Проверь, что контейнер поднялся и порт именно 5433:
`docker compose -f docker-compose.dev.yml ps`.

**Приложение стартует и сразу падает на ключе.** Значит `AUTH_JWT_PRIVATE_KEY` пустой или
переносы строк потерялись. Проще всего перегенерировать командой из шага 3.

**Порт 8080 занят.** Поменяй `APP_URL` в `.env`, например на `0.0.0.0:8081`.

**Тесты падают через раз.** Запусти с `-p 1`, см. раздел про тесты.
